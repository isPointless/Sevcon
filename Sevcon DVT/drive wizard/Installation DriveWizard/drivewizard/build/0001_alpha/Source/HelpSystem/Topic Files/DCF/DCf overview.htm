Device configuration Files (DCFs)

A DCF file is a snapshot of part of a CAN node's object dictionary incluing the actual values contained in this Object Dictionary.
You may wish to clone some of the settings on one CAN node to download them to a another connected CANopen node or a CANopen node on a different vehicle. Alternatively you may wish to back up some of the settings non a CANopen node.
To do this Drive Wizard provides a method of automatically creating, opening, checking and Downloaing DCF files to connected CANopen devices.

  • Creating a DCF store
  • Saving the DCF store to file
  • Populating the DCF store from a file
  • Downloaing the DCF store to a connected CANopen node
  • Modifying the identity of the DCF store

    • Creating a DCF Store

    To create a DCF store just drag the parts of the Object Dictionary that you want to clone from a source CANopen node to the DCF icon at the bottom of the Structure Pane. As you drag this item you should see a ghost string of ht object (if not click and drag a gain slightly slower) As the object locates over the DCF icon changes and the cursor changes to a copy cursor. You can now drop the Source Object Dictionary item into the DCF store. Drive Wizard will automatically add a copy of the source icon and all its underlying Object Dictionary items into the DCF store.
    You can add as may icons for a single CAN node source as you like. Drive Wizard will automatically detect and prevent re-addition of duplicate items and addition of items from more than one CANopen node.

    If you wish to remove items from the DCF store then right click on the branch that you wish to remove to simultaneously select the branch and to display the context menu

    This will remove all Object Dictionary items under this branch from the DCF store

    Saving the DCF stroe to file

    Once you have filled you DCF store then you can either save the contents to a DCF file for later use or download the associated Object Dictionary values to another connected device of the same type. to save the DC store to file just highlight any icon in the DCF branch and select the save option from the DCF context menu.

    Downloaing DCF store to a connected CANopen node

    For safety reasons, the contents of the DCF store can only be downloaded to a CANopen device with the same Vendor ID, product cod and revision number. Drive Wizard automatically determines and displays only CANopen nodes that meet this constraint.

    Populating the DCF store from a file

    To open a DCF file right click on any icon in the DCF branch and and opt to populate the DCF store from file. This will empty the current DCF store and refill it from the selected file. You can then modify, download and save this DCF store as normal.

    Modifying the identify of the DCF store

    Occasionally it is necessary to download an existing DCF file to a non-matching CANopen node. For example if the revision number has changed but the items in the DCF file are unaffected by this change. To do this you need to associate the new EDS file with the existing DCF file. This will change the product code and revision number in the DCF file accoringly.

     

    First populate the DCF store by either dragging items in or by opening an existing DC file. Now highlight the DCF icon to display the Associate new EDS file with this DCF control. Select the new EDS using the drop down lists. Now save the DCF to file. Drive Wizard will ask you to confirm that you wish to change the EDS reference.

    		 Changing individual parameters can affect vehicle perfromance and safety and should only be attempted by expert users. Use Drive Wizard guided processes to change the vehicle set up.
