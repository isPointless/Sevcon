CAN and CANopen: overview

Object Dictionary

All CANopen devices maintain an area of memory, parts of which are accessible to other CANopen devices via the CAN bus. This area of memory is known as the Object Dictionary

The CAN node's Object Dictionary is divided into parameters of variable type and length e.g. 32 bit unsigned integers, 64 bit Real numbers, Unicode strings.


To extract individual parameter from the Object Dictionary it is necessary to have a description of the type, position and length of each parameter. This description is known as the Electronic Data Sheet or EDS. each CAN node type needs to have its own EDS.

Matching an EDS file to an Object Dictionary.

A few mandatory Object Dictionary parameters have data types, position and length pre-defined by CANopen specification. These parameters can therefore be reliably read even without an EDS file.

One of these is known s the Identity Object. Drive Wizard uses three of its sub-objects to determine whether an EDS file is the matching EDS file for the CAN node's object Dictionary. These sub-objects are:

The CiA Vendor ID.
This is a unique 32 bit number assigned by the CiA to each registered CANopen vendor. Sevcon's vendor ID is.30.

The product code.
This is a 32 bit number sued to identify what type of device this CAN node is

The revision number.
This is a 32 bit number used to identity the revision of this CAN node e.g. an existing CAN Node could be upgraded and its object dictionary changed as part of this upgrade. In these circumstances the product code may stay the same but the revision number would be incremented.

Drive Wizard requires that the Vendor ID, product code and revision number ALL match for it to associate an EDS file with a connected CAN node's Object dictionary.

Device Configuration files (DCFs)

A specialized variant of the EDS file can be created known as a DCF. The major differences between an EDS and a DCF are that the DCF file contains actual values for Object dictionary parameters and the DCF contains a back reference to the EDS on which it was based. Drive Wizard can create DCF files for any connected CANopen node. See DCF
