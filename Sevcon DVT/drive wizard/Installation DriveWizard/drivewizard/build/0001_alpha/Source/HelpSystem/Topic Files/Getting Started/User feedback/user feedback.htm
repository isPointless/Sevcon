User Feedback

User feedback is provided by the following screen items :

  • Status bar
  • Progress bar
  • Drive Wizard busy Indicator
  • User Instructions label
  • System Structure icons

Status bars

All Drive Wizard screens contain a status bar. The status bar on secondary windows and the rightmost panel of the main screen status bar tells you what action Drive Wizard is currently perfroming. On completion of an action, an indication of success/failure is displayed in this panel.

The Status bar in the main screen contains two more panels

The leftmost panel displays the current baud rate and whether Drive Wizard is using real or Virtual CNA nodes

The middle panel displays the System login level. This is the Login level used for System wide Guided Processes and is the lowest login level achieved on any connected Sevcon node that is running application software.

System Structure icons

The following key shows all possible icons that can be displayed in the System Structure (displayed in the Structure Pane) . Some guided process require minimum user login levels and specific vehicle node conditions to be perfromed. Drive WIzard automatically displays/hides Guided Process icons as appropriate.

Icon relating to the vehicle
The root icon indicating the connected vehicle.
Start Guided Process to change the baud rate and CNA node IDs of connected CNA nodes
Start Guided Process to configure the automatic messages (System PDOs) transmitted across the CAN bus

 

Icons relating to a single CAN node
Represents a CAN node that is connected to the vehicle bus. NMT master node shave a red outline, all others nodes have a dark gray outline.
A flashing orange LED indicates that the node is in NMT pre-operational state, constantly on orange LED denotes a node in the operational state. All other NMT state are represented with the orange LED off.
The green LED represents the fault status of the node. For Sevcon nodes this LED mimics the status LED on the controller itself , a constantly on green LED indicates normal operation. If a fault is active then the flash rate indicates the nature of the fault (defined here). For non-Sevcon nodes the green LED is set to on unless the node transmits an emergency message when the green LED will be flashed.
Represents a multiple sections of the Object Dictionary. These groupings are included to improve readability. Drive WIzard does not read parameter values from the node in response to selecting multiple group icons.
Represents a single section of the Object dictionary. This can be a CANopen defined section or a section defined by function (Sevcon nodes) . When you first select this icon after connection Device Wizard will read and display the current value for each Object Dictionary parameter in the section
Represents a single Object Dictionary item that contains sub items.
Represents a single Object Dictionary sub item or item with no associated sub items.
Start Guided Process to perfrom Self Characterization on the associated motor.
Start Guided Process to reprogram this CAN node
Start Guided Process to link the physical Inputs/Outputs on a Sevcon CAN node to items in its Object Dictionary (Internal (or virtual) PDOs)
Start Guided Process to display and edit a Sevcon event log
