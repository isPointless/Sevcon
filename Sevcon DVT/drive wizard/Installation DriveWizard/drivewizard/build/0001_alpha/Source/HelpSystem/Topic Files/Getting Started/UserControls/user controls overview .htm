Context Menus
Context menus are used to show options that relate to the currently selected screen item. Context menus display all the processes that can be applied to this screen item. When you right click on a screen item its context menu (if available) is displayed. Some context menu options are dependent on User login level and vehicle mode. Drive Wizard automatically updates the context menus and hides/disables actions that cannot currently be perfromed.

Drag and Drop icons
Where a task involves linking one or more Object Dictionary items to a function e.g. adding items to the DCF store or mapping Object Dictionary items into System PDOs, Drive WIzard enables Drag&Drop of the icon from the System structure to its destination. To drag an icon, click and hold the left mouse button over the icon,. With the mouse button held down, drag the item to it destination. A shadow copy of the icon's text will be dragged to indicate that you have successfully dragged the item. When the mouse is over a suitable target, Drive Wizard changes the mouse cursor to a copy symbol and highlights the destination.

Window GUI controls

General
Drive Wizard hides or disables controls that are not relevant to the current user level and vehicle mode. When either the user login level or vehicle mode changes Drive Wizard reevaluates which controls to show/enable.
Generally, when Drive Wizard is busy user controls to initiate other tasks are disabled until Drive Wizard has finished. Exceptions to this include

Close Buttons
Always available but during some critical tasks Drive Wizard will display a warning and ask you confirm window closure

Drive Wizard Exit menu Option
Always available but during some critical tasks Drive Wizard will display a warning and ask you confirm exit

Parameter Table.

Changing individual Object Dictionary parameters can affect vehicle perfromance and safety and should only be attempted by expert users. Use Drive Wizard guided processes to change the vehicle set up.

The Parameter Table is displayed in the Detail Pane and allows you to enter new values for Object Dictionary items. Drive Wizard allows prevents you from editing parameters that are read only for the current user and vehicle mode. Parameters that can be edited ar always show with a blue background. Non editable parameters have either a white or mauve ( heaing lines) back ground. The parameters are color code to make it clear which items can/cannot be edited. If the user login level or vehicle mode changes then Drive WIzard automatically updates which items are now editable. Entering values into the Parameter Table
When you click on an editable Object Dictionary parameter values Drive Wizard checks whether this parameter has an associated list of valid values If so, Drive WIzard automatically displays a drop down list using descriptive names for the valid values These drop down lists ar also used in table associated with Drive Wizard logs

If a parameter has no associated list then Drive Wizard waits for you to enter the new value. Drive Wizard then checks the entered value. Drive Wizard checks for and report the following error types.

Range checking error
Occurs if the entered value. is greater than the value in the Max column or less than he value. in the Min column

Format checking
Occurs if a hexadecimal parameter is entered without the leading "0x".

 

Typographical errors
Occurs if an entry fromat doe snot match the CANopen data type e.g. a letter was entered when Drive Wizard was expecting a number.

If Drive WIzard reports and error then a symbol is displayed in the offening cell and an explanation of the error is displayed as a tool tip. you should re-enter the value. or press Escape or Delete to redisplay the original value.
