CANopen terms

EDS

Electronic Data Sheet. A file that describes the type, size, location and name of every parameter in a CANopen node's Object dictionary. Optionally minimum and maximum limits and a default valaue can be defined for parameters. Comments can be includeded. Sevcon EDS files contain addiotnal comment based fields marked 'SEVCONFIELD' to provide addtional functionaltiy.

DCF

Device Configuration File. A file based on EDS format that gives a vlaue for each Objrect Dictionary parameter contianed. A refernce to its source EDS file is included.

Object Dictionary

A area onf memory on a CANopen node that can be accessed by other CANopen nodes. The key to accessing specific parameters within this Object Dcitionanry is provided by the CANopen node's EDS file

Drive Wizard Data Structures

Parameter Table
Drive WIzard stores a Parameter table for each connected CANopen node. The Parameter Table combines information form the EDS and the node. when yo click on a icon in the structure view that represents part of the Object Dictionary that Drive Wizard applies a row filter to the Parameter Table to show only these Object Dictionary items that sit under the selected icon. A column filter is applied to the Parameter Table dependent on the the current user access level The aim is to show only the columns that you are interested in.

System Structure
The data and Guided Processes associated with both the vehicle and individual connected nodes organized into a Tree View and displayed in the Structure Pane.
